1. Introduction

A common problem when exchanging CoRAL [I-D.ietf-core-coral] documents is aligning the producing and consuming endpoints with regards to the vocabulary and the associated compression dictionary used both in abstract and on the wire.Â¶

For example, entities that consume CoRE problem details [I-D.ietf-core-problem-details] will typically need to understand the semantic associated with the compressed elements (in particular, the problem type code) in order to make sense of the received document.Â¶

This memo proposes data formats and protocol interactions that allow:Â¶

  • Producing endpoints to advertise a vocabulary used when interacting with a certain resource (Section 3.1);Â¶
